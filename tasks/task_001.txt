# Task ID: 1
# Title: Plugin Initialization and Structure Setup
# Status: pending
# Dependencies: None
# Priority: high
# Description: Create the basic plugin structure with main file, readme, and directory organization following WordPress plugin standards.
# Details:
1. Create main plugin file with standard WordPress header
2. Set up plugin activation/deactivation hooks
3. Create directory structure (includes/, admin/, public/, assets/)
4. Register autoloader for classes
5. Define plugin constants

```php
<?php
/**
 * Plugin Name: PizzaPilot
 * Description: A WordPress plugin for pizza delivery businesses to manage time slots and delivery logistics.
 * Version: 1.0.0
 * Author: Business in a Pizza Box
 * Text Domain: pizzapilot
 * Requires at least: 5.0
 * Requires PHP: 7.2
 * WC requires at least: 3.0
 * WC tested up to: 7.0
 */

// Exit if accessed directly
if (!defined('ABSPATH')) exit;

// Define plugin constants
define('PIZZAPILOT_VERSION', '1.0.0');
define('PIZZAPILOT_PLUGIN_DIR', plugin_dir_path(__FILE__));
define('PIZZAPILOT_PLUGIN_URL', plugin_dir_url(__FILE__));

// Include the main plugin class
require_once PIZZAPILOT_PLUGIN_DIR . 'includes/class-pizzapilot.php';

// Activation/Deactivation hooks
register_activation_hook(__FILE__, array('PizzaPilot', 'activate'));
register_deactivation_hook(__FILE__, array('PizzaPilot', 'deactivate'));

// Initialize the plugin
function run_pizzapilot() {
    $plugin = new PizzaPilot();
    $plugin->run();
}
run_pizzapilot();
```

# Test Strategy:
1. Verify plugin activates without errors in WordPress
2. Check plugin appears in WordPress plugins list
3. Confirm no PHP errors or warnings on activation
4. Test with both latest WordPress version and minimum required version
5. Verify WooCommerce dependency check works correctly
